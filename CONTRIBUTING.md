# Contributing Guidelines

Thank you for your interest in contributing to ``awslabs/startups``.

## RFCs for New Features, Artifacts, and Major Changes

For **new features**, **new artifacts** or **major changes**, contributors must first open an
[RFC (Request for Comments)](https://github.com/awslabs/startups/issues/new/choose) issue before
doing any work.

This ensures public visibility and allows maintainers to review proposals
before implementation begins. 

**RFC titles must start with `RFC:` prefix**, e.g. `RFC: Add support for X`.

---

## Code of Conduct

This project has adopted the [Amazon Open Source Code of Conduct](https://aws.github.io/code-of-conduct). See the [Code of Conduct FAQ](https://aws.github.io/code-of-conduct-faq) or contact opensource-codeofconduct@amazon.com with questions.

---

## Security Issue Notifications

If you discover a potential security issue, notify AWS/Amazon Security via the [vulnerability reporting page](http://aws.amazon.com/security/vulnerability-reporting/). Do **not** create a public GitHub issue.
